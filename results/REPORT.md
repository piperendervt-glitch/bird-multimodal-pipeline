# マルチモーダル鳥種分類パイプライン — 統合報告書

## プロジェクト概要

CAGL（Consensus-Agreement Gate Learning）の新規性検証から出発し、
マルチモーダルAIモデル統合アーキテクチャの設計・実装・検証を行った。

リポジトリ: cagl-synthetic（将来的に bird-multimodal-pipeline にリネーム予定）
環境: Windows 11, NVIDIA RTX 5060 Ti (16GB, Blackwell sm_120),
      PyTorch 2.12.0.dev+cu128 (nightly)

---

## 実験結果一覧

### CAGL 基礎検証（完了・アーカイブ済み）

| 環境 | V4 効果量 (d) | 結論 |
|------|-------------|------|
| 合成データ | +9.58 | 再現 |
| sklearn 実データ | ~0 | 効果なし |
| MMLU-Pro キャッシュ | +0.612 | 効果極小 |
| MMLU-Pro ローカル LLM | -0.424 | 効果なし |

**結論:** CAGLのnoveltyは合成データ固有。実世界での実質的意義なし。

### Phase 1: DINOv2 単独（CUB-200-2011, 200種）

| 手法 | Top-1 | Top-5 | F1 |
|------|-------|-------|-----|
| k-NN (k=5) | 85.11% | 94.91% | 0.8499 |
| LogReg | **87.31%** | 97.69% | 0.8730 |
| MLP (hidden=128) | 86.61% | 97.10% | 0.8647 |

**発見:** DINOv2 特徴量は線形分離性が高く、MLP の非線形性が不要。

### Phase 2: 色彩 + 形態特徴の追加

| 特徴量 | LogReg Top-1 | MLP Top-1 |
|--------|-------------|-----------|
| DINOv2 のみ | 87.73% | 87.12% |
| DINOv2 + 色彩 | 87.21% | 86.95% |
| DINOv2 + 形態 | 87.54% | 86.80% |
| DINOv2 + 色彩 + 形態 | 87.02% | 86.78% |
| 色彩 + 形態のみ | 3.81% | 4.26% |

**発見:** 同一モダリティ（画像）からの手作り特徴は DINOv2 に情報を追加しない。

### Phase 3: DINOv2 + BirdNET（異なるモダリティ統合）

5種の特定種検出タスク（2値分類）:

| 手法 | 平均 F1 |
|------|---------|
| DINOv2 単独 | 0.9507 |
| BirdNET 単独 | 0.5268 |
| DINOv2 + BirdNET + LogReg | **0.9616** |
| DINOv2 + BirdNET + MLP | 0.9422 |

**発見:** 異なるモダリティ（画像+音声）の統合は相補性を持ち有効（+1.09pp）。

### Phase 3b: 音声データ増量

| 手法 | Phase 3 (20件/種) | Phase 3b (50-100件/種) |
|------|-------------------|----------------------|
| DINOv2+BirdNET+LogReg | 0.9616 | 0.9471 |
| DINOv2+BirdNET+MLP | 0.9422 | 0.9453 |

**発見:** MLP vs LogReg の差が 1.94pp → 0.18pp に縮小。データ量増加で MLP が改善。

### Phase 4a: YOLO 鳥検出

| 指標 | 値 |
|------|-----|
| 検出率 | 98.15% |
| 平均 IoU | 0.8756 |
| 切り抜き後精度 | 85.80%（元画像 87.31% より -1.52pp） |

**発見:** DINOv2 は背景コンテキストも種の判別に使用している。切り抜きは精度を下げる。

### Phase 4b: 動画パイプライン（WetlandBirds, 13種, 178動画）

| 手法 | Accuracy | F1 |
|------|----------|-----|
| 映像のみ (DINOv2) | **96.30%** | 0.9590 |
| 音声のみ (BirdNET) | 25.93% | 0.2103 |
| 映像 + 音声 統合 | 96.30% | 0.9590 |

**発見:** 音声統合は害にならなかった（±0pp）。映像が支配的。鳴き声がない動画が大部分。

### Phase 4b-E: 高確信度動画分析

- 鳴き声がある動画（19/178）でも音声は精度に寄与せず
- 人間の確認で「映っている鳥 ≠ 鳴いている鳥」であることが判明
- カメラトラップ映像では映像と音声の対応が保証されない

### Phase 5e: 時間方向の平滑化（WetlandBirds）

| 手法 | Accuracy | 142-mallard |
|------|----------|-------------|
| フレーム平均（Phase 4b） | 96.30% | ✗ |
| 多数決 | **100.00%** | ✓ |
| 確信度加重投票 | 96.30% | ✗ |
| 指数減衰 | 92.59% | ✗ |
| スライディング窓 (3) | **100.00%** | ✓ |
| スライディング窓 (5) | **100.00%** | ✓ |
| 時間 Gate (CAGL 類似) | 92.59% | ✗ |
| YOLO 確信度加重 | **100.00%** | ✓ |

**発見:** 投票ベースの平滑化は確率ベースより頑健。高確信度の誤予測に影響されない。

### Phase 5f: VB100 検証（100種, 1416動画）

| 手法 | Accuracy | F1 |
|------|----------|-----|
| フレーム平均 | **89.05%** | 0.8823 |
| 多数決 | 86.93% | 0.8433 |
| YOLO 確信度加重 | 86.93% | 0.8483 |
| 先行研究 (ResNet-50) | 84.00% | — |

**発見:** 先行研究を +5.05pp 上回った。しかし 100 種ではフレーム平均が平滑化より優れる（WetlandBirds と逆）。

### Phase 5g: Great Tit 鳴き声分析（109,963曲）

| タスク | クラス数 | LogReg Top-1 | k-NN Top-1 |
|--------|---------|-------------|------------|
| ソングタイプ分類 | 725 | **99.20%** | 99.05% |
| 個体識別 | 360 | **98.48%** | 98.44% |

**発見:** 特化した埋め込み + LogReg で 725 クラスでも 99.2%。汎用 DINOv2 の 200 クラス 87.3% を大きく上回る。

### Phase 6: 動的適応（AdaptiveFeatureRouter）

3種類のルーター（LogReg適応、RandomForest適応、Hybrid）と
ヒューリスティックルーター（学習不要）を両データセットで評価。

WetlandBirds (13種, 27動画):

| 手法 | Accuracy |
|------|----------|
| 固定 多数決 / YOLO加重 / スライディング窓 | 100.00% |
| 適応ルーター（LogReg / RF / Hybrid） | 100.00% |
| ヒューリスティックルーター | 96.30% |
| 固定 フレーム平均 / 確信度加重 | 96.30% |

VB100 (100種, 283動画):

| 手法 | Accuracy |
|------|----------|
| 全固定手法 | 86.93% |
| 適応ルーター（LogReg / RF / Hybrid） | 86.93% |
| ヒューリスティックルーター | 86.93% |

**発見:**
- 適応ルーターは固定手法と同等。283動画中、手法で結果が分岐するのは7動画（2.5%）のみ。
- 改善余地が極めて少ない状況では動的選択の効果が出ない。
- ルーティング特徴量の重要度分析では確信度統計（conf_median, conf_max, conf_skewness）が上位を占め、Phase 5eで発見した「高確信度誤予測」問題と整合。
- ヒューリスティックルーター（学習不要）が学習型ルーターと同等の精度を達成。学習データ不足の状況では実用的な代替手段。

**注記:** 適応ルーターの評価はテストセット上のK-fold CV（K=5）で実施。
独立した学習/テスト分割ではないためバイアスあり。参考値として扱う。

## 主要な学び

### 1. モダリティの多様性が重要（Phase 2 vs Phase 3）
同じ画像から複数の特徴を抽出しても改善しない。異なるセンサー（カメラ + マイク）からの情報統合が有効。

### 2. DINOv2 は背景を使う（Phase 4a）
切り抜きで背景を除去すると精度が下がる。「枝に止まっている」「水辺にいる」等の環境情報も種の判別に寄与。

### 3. 統合方法は種数に依存する（Phase 5e vs 5f）
13 種では多数決が最良。100 種ではフレーム平均が最良。固定的なルールは汎用的に使えない。

### 4. 映像と音声の対応が保証されない（Phase 4b-E）
カメラトラップでは「映っている鳥」と「鳴いている鳥」が異なる。音声統合が機能する条件は限定的。

### 5. 特化した埋め込みの力（Phase 5g）
汎用モデル（DINOv2）より特化モデル（Great Tit 埋め込み）の方が圧倒的に高精度。ただしスケーラビリティとのトレードオフ。

### 6. LLM のショートカット問題（全体を通じて）
合成データの有利設計、分類器の弱体化、LLM 予測の知覚モデル代替。
8項目テンプレート（NeurIPS/ICML チェックリスト + LLM 特化の失敗条件・ショートカットチェック）で対処。

### 7. 動的適応の効果は改善余地に依存する（Phase 6）
パイプラインの精度が十分に高い場合、集約方法の選択で結果が変わる動画は極少数。
動的適応が効果を発揮するのは「手法によって結果が分かれる」状況が多い場合に限られる。
